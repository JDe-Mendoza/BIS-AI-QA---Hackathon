The one thing a judge watches end-to-end

An admin builds a Microlearning topic with real content, a learner opens that content, and completion is tracked — the topic ticks 2/3 → 3/3 → Completed.

Committing to — finished to ~90%The vertical slice
  • Admin — build content
  • Turn Microlearning on in Setup; the module tile + end-user tab appear
  • Dashboard: topic cards (grid + list), status pillbox, search, Add Topic, empty state
  • Create a topic — title (char count + validation), description, default image
  • Topic Content page — header, collapsible description, Add Content, grid/list, empty state
  • Add content of type Link, PDF, Video (URL embed + uploaded MP4) and SCORM "Activity" — with type icon + last-updated
  • Edit a content item (incl. edit-only "Complete Again")
  • Bulk-select content — checkboxes + action bar (deactivate / reactivate / purge)
  • Topic Settings — general metadata (title / image / description)
  • Multi-language — portal-language default + additional languages, per-language Title / Description / Source, with auto-translate (create + Manage Languages)
  • Manage & lifecycle
  • Image upload + crop modal (topic + content) — default image as fallback
  • Draft topic status — pauses learner access
  • Deactivate / Reactivate + Deactivated view (single & bulk)
  • Purge — permanent delete with purge-code confirm
  • Drag-to-reorder content (6-dot handle)
  • Notes / audit trail on topic actions
  • Decommission topic state (beyond Deactivate)
  • Access — assign topics
  • Grant access via permissions — By User and By Company Role (Add Permission UI on the topic)
  • Learner — consume content
  • Homepage: granted topics with New / In Progress / Completed, All/In Progress/Complete filter, search
  • Open a topic and see its content
  • Open a Link / PDF / Video / SCORM item → completion recorded (incl. SCORM completion detection), progress updates
  • Email + SMS notifications to learners (topic assigned / updated / reminders)
  • Home-page Training section — surfaces assigned / in-progress / completed microlearning on the portal home
  • Mobile — SafeTapp
  • Mobile app (React Native): browse topics from the Training tab + open all content types in-app (Link / PDF / Video / SCORM), completion synced
  • Quality
  • Automated tests ship: topic CRUD, content CRUD, completion tracking — the swb done gate
Cut from the case — provision only, don't buildPre-empts the critique
  • The big cuts
  • Events scheduling engine — frequency/cadence scheduling + custom email editor (the notifications themselves are in scope; the scheduler is not)
  • Anything AI — Generate-Using-AI topic/content (removed entirely, not even a greyed "coming soon")
  • Feature depth
  • By Location permission type — By User + By Company Role cover the demo
  • CEUs (Continuing Education Units)
  • Public / network topics, Launch to Companies
Never cut: create topic → add content → learner opens → completion tracked. Inside-slice fallbacks if a lane slips: list view → learner search → drop By-Company-Role (keep By-User) → 3rd content type. Rubric bet: realistic & ~90% done beats inflated & half-done.
